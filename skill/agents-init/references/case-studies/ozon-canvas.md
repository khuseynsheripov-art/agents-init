# Ozon Canvas Case Study

Use this only as a pressure-test example for context-referenced workflow drift. Do not copy the product direction into other projects.

## Failure Pattern

The user expected the work to examine how an Ozon image-set workflow would fit into the existing Canvas `/image` capability. Later planning translated that into a temporary sidecar/workbench route. The user had not read every T4/T5 document, so "Option A" did not communicate the visible consequence.

## General Lesson

The failure was not just ambiguity or a missing keyword. The main agent should have recovered anchors, synthesized the prior intended mainline against the current artifact, and asked the upstream contradiction first.

```text
Recovered mainline:
- plugin/source package -> existing Canvas image workflow -> suite/manifest/export path

Current observed artifact:
- temporary sidecar/workbench route

Contradiction:
- the visible slice may prove workflow readability, but it does not prove existing-surface integration or user acceptance of the new route

First confirmation:
- "Are you saying we should return to the existing Canvas insertion plan before continuing the sidecar?"
```

## Consequence Disclosure

Before asking the user to approve a similar option, disclose:

- `visible_change`: which route, page, panel, output, or workflow appears first;
- `integration_mode`: core integration, temporary sidecar/workbench, adapter, import/export path, or analysis-only;
- `what_yes_authorizes`: the exact work unlocked by the answer;
- `what_yes_does_not_authorize`: UI acceptance, export readiness, seller-ready claims, generated image quality, or later gates still blocked;
- `rejected_or_deferred_paths`: direct existing-surface integration or other paths parked;
- `rollback_or_reframe`: how to return to the upstream insertion plan if the visible slice feels wrong.

## What This Case Does Not Prove

- It does not prove the sidecar was wrong.
- It does not prove the existing surface is always the correct product direction.
- It does not prove backend tests or route-intent output can replace user-visible acceptance.
