# Release And Sanitization Plan

## Source Of Truth

This repository tracks the installable `agents-init` skill under `skill/agents-init`.

The installed runtime copy is generated by `scripts/install-local.ps1` and should not be edited as the long-term source of truth once the repository workflow is adopted.

## Commit Scope

Commit by default:

- `skill/agents-init/**`
- `scripts/install-local.ps1`
- `scripts/sync-from-installed.ps1`
- `README.md`
- `VERSION`
- `.gitignore`
- sanitized release notes and docs

Do not commit by default:

- `.workflow/**` runtime state and receipts;
- `.tmp/**`;
- `maestro-knowledge-base/**`;
- raw model receipts with session ids, costs, or local account profiles;
- private business project state from Ozon or other worktrees;
- local absolute paths unless used as generic placeholders.

## Private First

Use a private GitHub repository while the skill is still tied to local workflow experiments. Public release requires a separate review:

- replace user/profile paths with placeholders;
- remove or generalize business cases;
- remove raw screenshots, receipts, and session ids;
- keep only sanitized pressure-test examples.

## Update Flow

1. Edit `skill/agents-init`.
2. Run quick validation.
3. Run project-template workflow validation.
4. Install locally with `scripts/install-local.ps1`.
5. Smoke an adopted project.
6. Commit and tag when stable.
